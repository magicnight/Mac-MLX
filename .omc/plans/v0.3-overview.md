# macMLX v0.3 — Overview

> v0.2.0 shipped 2026-04-17. v0.3 cycle begins.

**Goal:** Deliver the features that are autonomously implementable
(no external binaries, no Xcode UI user-action required) and close
the v0.2 carryover items that fit.

**Non-goals for v0.3:** SwiftLM / Python MLX engines (sandbox policy
review; #12 / #13 remain in v0.3+), Apple Developer notarization
(#19), MAS distribution.

## #22 Benchmark — detailed scope

### Data model (expand existing `BenchmarkResult`)

Existing (Stage 2) is minimal. Extend to match `.claude/features/benchmark.md`:

| Existing field | Keep | Reason |
|---|---|---|
| `id`, `modelID`, `engineID` | ✅ | identity |
| `promptTokens`, `completionTokens` | ✅ | inputs |
| `promptTPS`, `generationTPS`, `ttftMs` | ✅ | core metrics |
| `system: SystemInfo` | extend | add macOSVersion |
| `timestamp` | ✅ | |

New fields:
- `macOSVersion: String`
- `macMLXVersion: String` (from `Bundle.main`)
- `engineVersion: String` (from engine)
- `modelParams: String?`, `modelQuant: String?` (parsed from modelID)
- `memoryUsedGB: Double` (peak unified memory during run)
- `modelLoadTimeS: Double` (cold-load)
- `runs: Int` (how many measured iterations)
- `notes: String` (user-editable)

### New types

- `MacMLXCore/Managers/BenchmarkStore.swift` — actor; persists `[BenchmarkResult]` to `~/.mac-mlx/benchmarks/history.json`
- `MacMLXCore/Managers/BenchmarkRunner.swift` — actor; orchestrates warm-up + N measured runs, computes median, peak memory via `os_proc_available_memory` or `task_info`
- `MacMLXCore/Util/HardwareInfo.swift` — `sysctlbyname` wrapper for `machdep.cpu.brand_string` + `hw.memsize`

### SwiftUI

- `macMLX/macMLX/Views/Benchmark/BenchmarkView.swift`
- `macMLX/macMLX/Views/Benchmark/BenchmarkViewModel.swift` (@Observable @MainActor)
- Add `Benchmark` to `SidebarSection` (or whatever the sidebar enum is)
- Sections:
  1. **Config** — model picker, engine picker (disabled to MLX Swift in v0.3), prompt/gen token sliders, runs stepper
  2. **Last result** — big numbers for prefill / generation / TTFT / memory / load time. `[Share to Community]` button + `[Copy as JSON]`.
  3. **History** — `List` of prior runs; swipe-delete, tap-expand

### Sharing

- "Share to Community" builds a GitHub issue URL:
  `https://github.com/magicnight/Mac-MLX/issues/new?template=benchmark_submission.yml&data=<urlencoded JSON>`
- Requires an `.github/ISSUE_TEMPLATE/benchmark_submission.yml` to exist in the repo.
- Automated collector pipeline (GitHub Action → data.json → leaderboard page) is out of scope for v0.3; the feature spec explicitly defers it to v0.2 (now v0.3+). Manual collection for v0.3 launch.

### Tests

- `BenchmarkResultTests` — round-trip Codable
- `BenchmarkStoreTests` — save + load + list ordering, corrupt-file tolerance (same pattern as `ConversationStore`)
- `BenchmarkRunnerTests` — median aggregation logic (with a mock engine emitting canned samples)
- `HardwareInfoTests` — non-empty chip string, non-zero memory

### Build order

1. Extend `BenchmarkResult` model
2. Add `HardwareInfo` + tests
3. Add `BenchmarkStore` + tests
4. Add `BenchmarkRunner` + tests (with mock engine)
5. SwiftUI `BenchmarkView` + `BenchmarkViewModel`
6. Wire into sidebar navigation
7. Issue template file + share URL builder
8. Commit per step; single PR at end

## #20 Homebrew tap — detailed scope

- New repo: `magicnight/homebrew-macmlx`
- Formula: `Formula/macmlx.rb` — installs prebuilt `macmlx` binary from the v0.2.0 / v0.3.0 GitHub release
- Release workflow update: upload `macmlx-<version>-arm64.tar.gz` as a release asset (currently only DMG uploads)
- Users `brew tap magicnight/macmlx && brew install macmlx`

Smaller scope than #22. Can ship incrementally once the binary asset lands.

## Exit criteria for v0.3.0 tag

1. #22 Benchmark feature closed; history persists; share flow works
2. At least one of #16 / #20 / #23 closed or demoted to v0.4
3. `swift test --package-path MacMLXCore` green (60 + new)
4. `xcodebuild -scheme macMLX build` green
5. CHANGELOG `[Unreleased]` promoted to `[0.3.0] - YYYY-MM-DD`
6. Unsigned DMG same distribution as v0.2.0
