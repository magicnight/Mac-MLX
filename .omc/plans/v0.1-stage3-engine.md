# Stage 3 — MLXSwiftEngine

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Branch: `feat/stage3-engine`. Worktree-isolated.

**Goal:** Implement the default in-process inference engine using `mlx-swift-lm` (3.31.3) behind the `InferenceEngine` actor protocol from Stage 2.

**Architecture:** One actor type `MLXSwiftEngine` conforming to `InferenceEngine`. State machine: idle → loading → ready → generating → ready → idle. Wraps mlx-swift-lm's high-level chat API. No HTTP, no UI, no persistence.

**Tech Stack:** Swift 6 · `MLXLLM` (mlx-swift-lm 3.31.3) · `Foundation` · Swift Testing.

**Source spec:** `.claude/features/inference-engines.md` § Engine 1.

## Task 1: MLXSwiftEngine actor

Required public API (matches `InferenceEngine` protocol):

```swift
public actor MLXSwiftEngine: InferenceEngine {
    public let engineID: EngineID = .mlxSwift
    public private(set) var status: EngineStatus = .idle
    public private(set) var loadedModel: LocalModel?
    public let version: String   // e.g. "mlx-swift-lm 3.31.3"

    public init() { ... }

    public func load(_ model: LocalModel) async throws
    public func unload() async throws
    public nonisolated func generate(_ request: GenerateRequest) -> AsyncThrowingStream<GenerateChunk, Error>
    public func healthCheck() async -> Bool
}
```

### Behavior contract

- **load**: set status to `.loading(model: model.id)`, call mlx-swift-lm to load from `model.directory`, store the resulting container in a private `var modelContainer: <type>?`, set status to `.ready(model: model.id)`. On any error, set status to `.error(reason)` and rethrow as `EngineError.modelLoadFailed(reason:)`.
- **unload**: clear `modelContainer`, set `loadedModel = nil`, status to `.idle`.
- **generate**: must be `nonisolated` (returns stream synchronously). Validate that a model is loaded by reading isolated state via `Task { await self.loadedModel }` inside the stream's continuation closure. If no model: yield error `EngineError.modelNotLoaded` and finish. If model loaded: drive mlx-swift-lm's chat/streaming API, mapping each token chunk to `GenerateChunk(text: tokenText)`. Final chunk should include `finishReason: .stop` (or `.length` if maxTokens hit) and a populated `TokenUsage`.
- **healthCheck**: return `loadedModel != nil` if the engine considers itself responsive when a model is loaded. If a model isn't loaded, return `true` anyway (engine is responsive, just empty).

### Implementation notes

- The `nonisolated generate(_:)` is non-trivial because it can't `await` actor-isolated state directly. Pattern:
  ```swift
  public nonisolated func generate(_ request: GenerateRequest) -> AsyncThrowingStream<GenerateChunk, Error> {
      AsyncThrowingStream { continuation in
          Task { [weak self] in
              guard let self else { continuation.finish(); return }
              do {
                  try await self.runGeneration(request, into: continuation)
              } catch {
                  continuation.finish(throwing: error)
              }
          }
      }
  }

  private func runGeneration(_ request: GenerateRequest, into continuation: AsyncThrowingStream<GenerateChunk, Error>.Continuation) async throws {
      // actor-isolated; can read modelContainer, set status, etc.
  }
  ```

- Use `request.allMessages` to combine system prompt with user/assistant messages.
- Map `request.parameters.temperature`, `topP`, `maxTokens` to mlx-swift-lm's parameter type (whatever name it uses — see PF-2).

## Task 2: Tests

Don't load a real MLX model in unit tests — too slow and requires network. Instead test:

```swift
@Test
func newEngineStartsIdle() async {
    let e = MLXSwiftEngine()
    let s = await e.status
    let v = await e.version
    let id = await e.engineID
    #expect(s == .idle)
    #expect(id == .mlxSwift)
    #expect(v.contains("mlx-swift-lm"))
}

@Test
func loadingNonexistentDirectoryFailsWithEngineError() async {
    let e = MLXSwiftEngine()
    let bogus = LocalModel(
        id: "nonexistent", displayName: "X",
        directory: URL(filePath: "/tmp/definitely-not-a-model-\(UUID().uuidString)"),
        sizeBytes: 0, format: .mlx,
        quantization: nil, parameterCount: nil, architecture: nil
    )
    do {
        try await e.load(bogus)
        Issue.record("expected load to throw")
    } catch let err as EngineError {
        // Acceptable: modelLoadFailed or any EngineError variant
        switch err {
        case .modelLoadFailed: break
        default: Issue.record("unexpected EngineError: \(err)")
        }
    } catch {
        // mlx-swift-lm may throw its own error type — wrap in EngineError if not already
        Issue.record("expected EngineError, got \(type(of: error)): \(error)")
    }
    let status = await e.status
    if case .error = status { /* ok */ } else {
        Issue.record("expected status to be .error after failed load, got \(status)")
    }
}

@Test
func generateWithoutLoadedModelYieldsModelNotLoadedError() async {
    let e = MLXSwiftEngine()
    let req = GenerateRequest(model: "x", messages: [
        ChatMessage(role: .user, content: "hi")
    ])
    do {
        for try await _ in e.generate(req) { /* drain */ }
        Issue.record("expected stream to throw")
    } catch let err as EngineError {
        #expect(err == .modelNotLoaded)
    } catch {
        Issue.record("expected EngineError.modelNotLoaded, got \(type(of: error))")
    }
}

@Test
func healthCheckReturnsTrueWhenIdle() async {
    let e = MLXSwiftEngine()
    let healthy = await e.healthCheck()
    #expect(healthy == true)
}
```

If your implementation wraps mlx-swift-lm's error in `EngineError.modelLoadFailed(reason:)` consistently, the tests above pass without requiring a real model load.
