# macMLX v0.1 — Stage 1: Bootstrap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Three compilable targets — `MacMLXCore` SPM library, `macmlx` SPM CLI executable, `macMLX.app` SwiftUI Xcode project — wired together, with smoke tests passing locally and in GitHub Actions CI.

**Architecture:** Local SPM workspace with `MacMLXCore` shared between `macmlx-cli` (also SPM) and `macMLX` (Xcode project consuming local SPM). No Core domain logic yet — only placeholder types and version strings to prove the wiring works.

**Tech Stack:** Swift 6.3 · SwiftPM 6.3 · Xcode 26.4 · Swift Testing (`import Testing`) · GitHub Actions (macOS-15 runner) · `mlx-swift-examples` and `hummingbird` (declared as dependencies, not yet imported in code) · `swift-argument-parser` and `SwiftTUI` (declared for CLI).

## Task 5: GitHub Actions CI

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 5.1: Write the workflow**

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: macos-15
    timeout-minutes: 30

    steps:
      - uses: actions/checkout@v4

      - name: Select Xcode 16+
        run: |
          sudo xcode-select -s /Applications/Xcode_16.app
          xcodebuild -version

      - name: Show Swift version
        run: swift --version

      - name: Resolve & test MacMLXCore
        run: |
          swift package --package-path MacMLXCore resolve
          swift test --package-path MacMLXCore --parallel

      - name: Resolve & test macmlx-cli
        run: |
          swift package --package-path macmlx-cli resolve
          swift test --package-path macmlx-cli --parallel

      - name: Build macMLX.app
        run: |
          xcodebuild \
            -project macMLX/macMLX.xcodeproj \
            -scheme macMLX \
            -configuration Debug \
            -destination 'platform=macOS' \
            build CODE_SIGN_IDENTITY="" CODE_SIGNING_REQUIRED=NO
```

Caveats explained:
- `macos-15` runner is the most recent stable Apple-provided one. If GitHub publishes `macos-26`, switch to it (Xcode 26 is what we use locally).
- `Xcode_16.app` is the latest pre-installed version on `macos-15`. Adjust if Apple ships a newer one. Run `ls /Applications/ | grep Xcode` in a one-off workflow to discover what's available.
- We disable code signing in CI because we have no Apple Developer account in CI yet. Real signing comes in Stage 6.
