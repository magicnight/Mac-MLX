# Stage 3 — ModelLibraryManager + HFDownloader

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Branch: `feat/stage3-library`. Worktree-isolated.

**Goal:** Two actors that together let macMLX find local MLX models and pull new ones from Hugging Face Hub.

**Architecture:** `ModelLibraryManager` scans a directory and produces `[LocalModel]` using `ModelFormat.detect(in:)`. `HFDownloader` uses `URLSession` to (a) search the HF API for `mlx-community` models and (b) fetch model files into a destination directory with progress reporting. Pure Foundation — no Hummingbird, no MLX, no Pulse.

**Tech Stack:** Swift 6 · `Foundation` (URLSession, FileManager) · Swift Testing.

**Source spec:** `.claude/features/model-downloader.md`.

## Task 1: ModelLibraryManager

```swift
public actor ModelLibraryManager {
    public init(fileManager: FileManager = .default) { ... }

    /// Scan `directory` for subdirectories that look like MLX models and return them.
    /// Skips hidden dirs and dirs that don't match either MLX or GGUF heuristics.
    public func scan(_ directory: URL) async throws -> [LocalModel]

    /// Cached most-recent scan result; nil if `scan(_:)` not yet called.
    public private(set) var lastScan: [LocalModel]
}
```

### Behavior

- For each top-level subdirectory of `directory`:
  - Read its file listing
  - Call `ModelFormat.detect(in: filenames)`
  - If `.mlx`: build a `LocalModel` with `id = directoryName`, `displayName = directoryName`, `directory = subdirURL`, `sizeBytes = sum of all .safetensors file sizes`, `format = .mlx`. Try to extract `quantization` from the directory name suffix (regex `-(\d+bit)$`). Leave `parameterCount` and `architecture` nil for now (Stage 4+ enrichment).
  - If `.gguf`: skip (warn-log via simple `print` — Pulse arrives in settings sub-plan)
  - If `.unknown`: skip
- Return sorted by `displayName`.

### Tests

- Use `FileManager.default.temporaryDirectory.appending(path: "lib-test-\(UUID())")` as test root, populate with fake structures.
- `func scanFindsMLXLikeDirectory()` — create dir with `config.json`, `tokenizer.json`, dummy `model.safetensors`, expect 1 result with format `.mlx`.
- `func scanIgnoresGGUFAndUnknown()` — create dir with `model.gguf`, another with `random.bin`, expect 0 results.
- `func scanExtractsQuantizationFromName()` — dir named `Qwen3-8B-4bit`, expect `localModel.quantization == "4bit"`.
- `func scanReturnsEmptyForEmptyDirectory()` — fresh empty dir, expect `[]`.
- Cleanup temp dirs in `defer`.

## Task 2: HFDownloader

```swift
public actor HFDownloader {
    public init(urlSession: URLSession = .shared) { ... }

    /// Search Hugging Face Hub for mlx-community models matching `query`.
    public func search(query: String, limit: Int = 20) async throws -> [HFModel]

    /// Fetch the file list for a given model (e.g. "mlx-community/Qwen3-8B-4bit").
    public func files(for modelID: String) async throws -> [HFRemoteFile]

    /// Download a model into `directory/modelName/`. Calls `onProgress` after each file
    /// completes (no per-file streaming progress in v0.1). Returns the created directory URL.
    public func download(
        modelID: String,
        to directory: URL,
        onProgress: ((Double) -> Void)? = nil
    ) async throws -> URL
}

public struct HFRemoteFile: Codable, Hashable, Sendable {
    public let path: String        // e.g. "config.json", "model.safetensors"
    public let size: Int64?        // bytes, may be nil if HF doesn't expose it
    public let lfs: Bool           // true if served via LFS (large files)
}
```

### URL conventions (from spec)

- Search: `GET https://huggingface.co/api/models?author=mlx-community&search={q}&limit=20&sort=downloads&direction=-1`
- Model details: `GET https://huggingface.co/api/models/{modelID}` — response has `siblings: [{rfilename: String, size: Int?}]` for files.
- File download: `GET https://huggingface.co/{modelID}/resolve/main/{path}` — resolves LFS.

Decode all responses with `JSONDecoder.huggingFace` (added in Stage 2 to `HFModel.swift`).

### Behavior

- `search`: build URL via `URLComponents`, GET, decode `[HFModel]`, return.
- `files`: GET model details endpoint, decode envelope, map `siblings → [HFRemoteFile]`.
- `download`:
  1. Create `directory.appending(path: modelName)` (where `modelName` = last component of modelID after `/`).
  2. Call `files(for:)` to enumerate.
  3. For each file: build resolve URL, use `URLSession.download(from:)` async to stream to a temp file, then move into the model directory using `FileManager.moveItem(at:to:)`.
  4. After each file: call `onProgress(completedFiles / totalFiles)`.
  5. Return the model directory URL.
- Errors: throw `EngineError` doesn't quite fit; introduce a tiny new `DownloadError` enum in this file:
  ```swift
  public enum DownloadError: LocalizedError, Sendable {
      case badStatusCode(Int, url: URL)
      case writeFailed(String)
      case modelNotFound(String)
      public var errorDescription: String? { ... }
  }
  ```

### Tests

URL hitting is brittle in CI. **Mock URLSession via `URLProtocol`**:

```swift
final class MockURLProtocol: URLProtocol {
    nonisolated(unsafe) static var handler: ((URLRequest) -> (HTTPURLResponse, Data))?
    override class func canInit(with request: URLRequest) -> Bool { true }
    override class func canonicalRequest(for request: URLRequest) -> URLRequest { request }
    override func startLoading() {
        guard let handler = Self.handler else {
            client?.urlProtocol(self, didFailWithError: URLError(.badServerResponse)); return
        }
        let (response, data) = handler(request)
        client?.urlProtocol(self, didReceive: response, cacheStoragePolicy: .notAllowed)
        client?.urlProtocol(self, didLoad: data)
        client?.urlProtocolDidFinishLoading(self)
    }
    override func stopLoading() {}
}
```

Tests:
- `func searchDecodesMockResponse()` — set handler to return JSON list of 2 models, expect 2 `HFModel` results.
- `func filesDecodesSiblingsArray()` — handler returns `{"id":"x/y","siblings":[{"rfilename":"config.json"}]}`, expect 1 `HFRemoteFile`.
- `func searchPropagates404AsError()` — handler returns 404, expect `DownloadError.badStatusCode`.

Don't test real network in unit tests. Real download happens in Stage 3 Phase 3 integration test.
