# Stage 3 — SettingsManager + LogManager

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Branch: `feat/stage3-settings`. Worktree-isolated.

**Goal:** Persist user preferences as JSON at `~/.mac-mlx/settings.json`, and expose a `LogManager` that wraps Pulse's `LoggerStore` so the GUI Logs tab (Stage 4) can render with `PulseUI.ConsoleView`.

**Architecture:** Two independent actors. `SettingsManager` reads/writes a typed `Settings` struct to disk, with safe defaults if the file is missing or corrupt. `LogManager` is a thin shim over `Pulse.LoggerStore.shared` exposing categorical logging by `engine`, `inference`, `download`, `http`, `system`, `error`.

**Tech Stack:** Swift 6 · `Foundation` · `Pulse` 5.x · Swift Testing.

**Source spec:** `.claude/architecture.md` § Settings, `.claude/features/logs.md`.

## Task 1: Settings struct + SettingsManager

```swift
public struct Settings: Codable, Equatable, Sendable {
    public var modelDirectory: URL
    public var preferredEngine: EngineID
    public var serverPort: Int
    public var autoStartServer: Bool
    public var lastLoadedModel: String?
    public var onboardingComplete: Bool
    public var pythonPath: String?
    public var swiftLMPath: String?
    public var sparkleUpdateChannel: String       // "release" | "beta"
    public var logRetentionDays: Int

    public static let `default`: Settings = .init(
        modelDirectory: FileManager.default.homeDirectoryForCurrentUser
            .appending(path: "models"),
        preferredEngine: .mlxSwift,
        serverPort: 8000,
        autoStartServer: false,
        lastLoadedModel: nil,
        onboardingComplete: false,
        pythonPath: nil,
        swiftLMPath: nil,
        sparkleUpdateChannel: "release",
        logRetentionDays: 7
    )

    public init(...) { ... }
}

public actor SettingsManager {
    /// Production initialiser — uses ~/.mac-mlx/settings.json.
    public init()

    /// Test/preview initialiser — caller controls the file URL.
    public init(fileURL: URL)

    /// Snapshot of current settings.
    public private(set) var current: Settings

    /// Reload from disk. Falls back to .default on missing or corrupt file.
    @discardableResult
    public func reload() async throws -> Settings

    /// Mutate settings via a closure, then persist.
    public func update(_ mutate: (inout Settings) -> Void) async throws

    /// Replace settings wholesale and persist.
    public func replace(_ new: Settings) async throws
}
```

### Behavior

- File path defaults to `FileManager.default.homeDirectoryForCurrentUser.appending(path: ".mac-mlx/settings.json")`. Test initialiser allows pointing at a temp file.
- On `init()`: try reading the file. If missing → use `.default` and persist. If corrupt → use `.default` (do **not** overwrite the corrupt file; leave it for human inspection but proceed).
- Persistence: JSON pretty-printed (`outputFormatting = .prettyPrinted`), atomically written via `Data.write(to:options:.atomic)`. Ensure parent directory exists before writing.

### Tests

Use temp file URL for every test:
```swift
func makeTempSettingsURL() -> URL {
    FileManager.default.temporaryDirectory
        .appending(path: "settings-test-\(UUID().uuidString).json")
}
```

- `defaultsLoadedWhenFileMissing()` — pass a non-existent URL, expect `.current == .default`, file written.
- `roundTripPersistsAcrossInstances()` — make manager A, update port to 9999, make manager B at same URL, expect `current.serverPort == 9999`.
- `corruptFileFallsBackToDefaults()` — write garbage to the URL, init manager, expect `.current == .default`, original file untouched.
- `updateClosureMutatesAndPersists()` — call `update { $0.onboardingComplete = true }`, re-read raw file, decode, verify field set.

## Task 2: LogManager (Pulse wrapper)

```swift
import Pulse

public actor LogManager {
    public static let shared = LogManager()

    /// Backed by Pulse's shared LoggerStore (~/Library/.../pulse.sqlite by default).
    private let store: LoggerStore

    public init(store: LoggerStore = .shared) {
        self.store = store
    }

    public func log(
        _ message: String,
        level: LogLevel = .info,
        category: LogCategory = .system
    ) {
        store.storeMessage(
            label: category.rawValue,
            level: level.pulse,
            message: message,
            metadata: nil
        )
    }
}

public enum LogLevel: String, Sendable, CaseIterable {
    case debug, info, warning, error, critical
    var pulse: LoggerStore.Level {
        switch self {
        case .debug: return .debug
        case .info: return .info
        case .warning: return .warning
        case .error: return .error
        case .critical: return .critical
        }
    }
}

public enum LogCategory: String, Sendable, CaseIterable {
    case engine, inference, download, http, system, error
}
```

### Behavior

- Single `static let shared` for app-wide use; tests use a custom `LoggerStore` initialised with a temp directory:
  ```swift
  let url = FileManager.default.temporaryDirectory.appending(path: "log-test-\(UUID()).pulse")
  let store = try LoggerStore(storeURL: url)
  let manager = LogManager(store: store)
  ```
- The `pulse` mapping handles whatever Pulse's actual `Level` cases are — adjust if PF-1 reveals different names.

### Tests

- `logRecordsToBackingStore()` — log a message, then query `store.allMessages()` (or whatever Pulse 5.x exposes) and assert the message text and category appear.
- `logLevelsMapToPulseLevels()` — for each `LogLevel`, assert the resulting `pulse` value is the expected `LoggerStore.Level`.

If Pulse's testing API doesn't easily expose the message list, fall back to a simpler test: just verify `log(...)` doesn't throw or crash under various inputs.
