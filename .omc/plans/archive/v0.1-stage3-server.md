# Stage 3 — HummingbirdServer

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Branch: `feat/stage3-server`. Worktree-isolated.

**Goal:** OpenAI-compatible HTTP server on `localhost:{port}` that proxies an `InferenceEngine` over `/v1/chat/completions`, `/v1/models`, plus extension routes `/x/status` and `/health`.

**Architecture:** One `actor HummingbirdServer` that owns a Hummingbird `Application` instance plus a `weak`/strong reference to an injected `any InferenceEngine`. Routes are registered in init; server is started/stopped via `start(port:)` / `stop()`. Streaming `/v1/chat/completions` writes SSE chunks to the response body; non-streaming returns a single JSON object.

**Tech Stack:** Swift 6 · `Hummingbird` 2.22.0 · `Foundation` · Swift Testing.

**Source spec:** `.claude/api-contracts.md`.

## Task 1: HummingbirdServer

```swift
import Hummingbird

public actor HummingbirdServer {
    public init(engine: any InferenceEngine, settings: Settings)

    /// Listening port if running, else nil.
    public private(set) var listeningPort: Int?

    /// True iff the server has bound a port and is accepting connections.
    public var isRunning: Bool { listeningPort != nil }

    /// Start the server on the requested port; returns the actual bound port.
    /// May rebind to an adjacent port if `port` is busy (port + 1 .. port + 20).
    @discardableResult
    public func start(preferredPort: Int) async throws -> Int

    public func stop() async
}
```

### Routes

| Method | Path | Behavior |
|---|---|---|
| GET | `/health` | `200` `{"status":"ok"}` |
| GET | `/v1/models` | `200` `{"object":"list","data":[{"id":"<loadedModel.id>","object":"model","owned_by":"local"}]}` (or empty `data: []` if no model loaded) |
| GET | `/x/status` | `200` JSON with `status, loaded_model, memory_used_gb, memory_total_gb, uptime_seconds, requests_total, tokens_generated_total` |
| POST | `/v1/chat/completions` | OpenAI-style. If `stream: false`: collect all `GenerateChunk`s, return one `chat.completion` JSON. If `stream: true`: return `text/event-stream` with `data: {"id":..,"choices":[{"delta":{"content":...},"index":0}]}\n\n` per chunk and `data: [DONE]\n\n` at end. |
| POST | `/x/models/load` | `{model_path: String}` → engine.load(...) → `{"status":"loaded","model":id,"load_time_ms":N}` |
| POST | `/x/models/unload` | `{model: String}` → engine.unload() → `{"status":"unloaded"}` |

### Behavior contract

- Decode request bodies via `Codable`. Reject malformed bodies with `400` and OpenAI-shaped error envelope:
  ```json
  {"error": {"message": "...", "type": "invalid_request_error", "code": "..."}}
  ```
- For `/v1/chat/completions`: build a `GenerateRequest` from the OpenAI `ChatCompletionRequest`, call `engine.generate(request)`, transform output. **System prompt** comes from the first message with `role: "system"` if present.
- Track simple counters in actor state for `/x/status`: `requestCount`, `tokenCount`, `startedAt`. Memory values come from `MemoryProbe.totalMemoryGB()` — `memory_used_gb` can be `0.0` for v0.1 (we don't have a process-RSS reader yet).
- Port retry: if Hummingbird's `bind` throws on `preferredPort`, try `preferredPort + 1` ... `preferredPort + 20`. Throw `InferenceServiceError.noAvailablePort` if all 21 fail.

### Implementation notes

- The actor **owns** the Hummingbird Application object, but Hummingbird's `Application.run()` is long-running. Strategy: spawn a `Task` that calls `await app.run()` and store the `Task` so `stop()` can `cancel()` it.
- For SSE streaming, you'll need to write to the response body chunk-by-chunk. In Hummingbird 2, this is typically done via a streaming `ResponseBody`. Confirm the exact pattern in PF-1.
- Don't try to make the server `Sendable`-clean by declaring the engine `Sendable` — `any InferenceEngine` is already `Sendable` because `Actor` is `Sendable`.

## Task 2: Tests

Use `StubInferenceEngine` (already in `Tests/MacMLXCoreTests/Helpers/`) to feed deterministic chunks. Pick a free port via the OS (bind to `0` if Hummingbird supports it; otherwise pick a random high port and retry).

Tests:

```swift
@Test
func healthEndpointReturnsOk() async throws {
    let engine = StubInferenceEngine(engineID: .mlxSwift)
    let server = HummingbirdServer(engine: engine, settings: .default)
    let port = try await server.start(preferredPort: 19_000)
    defer { Task { await server.stop() } }

    let url = URL(string: "http://127.0.0.1:\(port)/health")!
    let (data, response) = try await URLSession.shared.data(from: url)
    let http = try #require(response as? HTTPURLResponse)
    #expect(http.statusCode == 200)
    let json = try JSONSerialization.jsonObject(with: data) as? [String: String]
    #expect(json?["status"] == "ok")
}

@Test
func modelsEndpointEmptyWhenNoModelLoaded() async throws { /* similar shape */ }

@Test
func modelsEndpointListsLoadedModel() async throws {
    let engine = StubInferenceEngine(engineID: .mlxSwift)
    try await engine.load(LocalModel(
        id: "test", displayName: "Test",
        directory: URL(filePath: "/tmp"),
        sizeBytes: 0, format: .mlx,
        quantization: nil, parameterCount: nil, architecture: nil
    ))
    /* start server, GET /v1/models, assert data[0].id == "test" */
}

@Test
func chatCompletionsNonStreaming() async throws {
    /* engine.load(stub model), POST /v1/chat/completions with stream:false,
       assert response is single JSON with choices[0].message.content == "stub-response" */
}

@Test
func portRetrySucceedsWhenPreferredPortBusy() async throws {
    let blocker = HummingbirdServer(engine: StubInferenceEngine(engineID: .mlxSwift), settings: .default)
    let port = try await blocker.start(preferredPort: 19_100)
    defer { Task { await blocker.stop() } }

    let server2 = HummingbirdServer(engine: StubInferenceEngine(engineID: .mlxSwift), settings: .default)
    let actualPort = try await server2.start(preferredPort: port)
    defer { Task { await server2.stop() } }
    #expect(actualPort != port)
    #expect(actualPort > port && actualPort <= port + 20)
}
```

Test cleanup: every test that starts a server **must** stop it in `defer` to release the port. Use unique high ports per test to avoid interference under `--parallel`.
