# macMLX v0.1 — Stage 4: SwiftUI Surfaces

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Each surface is its own task block; build them in order.

**Goal:** Wire `MacMLXCore` actors into 5 SwiftUI surfaces inside `macMLX/macMLX/`, producing a launchable menu-bar app where the user can complete onboarding, browse and download models, chat with a loaded model, and tweak settings.

**Architecture:** All views live in `macMLX/macMLX/Views/<Feature>/` per `architecture.md`. State lives in `@Observable` view models that own actor references. SwiftUI Previews use stub data (`StubInferenceEngine`, fake `LocalModel` arrays) — no real model load in Previews. Real engine wiring happens at app entry in `macMLXApp.swift`.

**Tech Stack:** Swift 6 strict concurrency · SwiftUI (macOS 14+) · `@Observable` macro · `NavigationSplitView` · `NSStatusItem` + `NSPopover` for menu bar · `MacMLXCore` from Stage 2/3 · `PulseUI` for the Logs view (Stage 4 imports `PulseUI` product into the Xcode target).

## Task 1: AppState + EngineCoordinator (foundation, no UI yet)

Build the model layer the surfaces will share.

**Files:**
- Create: `macMLX/macMLX/App/AppState.swift`
- Create: `macMLX/macMLX/App/EngineCoordinator.swift`

```swift
// AppState.swift
@Observable
@MainActor
public final class AppState {
    public let settings: SettingsManager
    public let library: ModelLibraryManager
    public let downloader: HFDownloader
    public let logs: LogManager
    public let coordinator: EngineCoordinator

    public init() {
        self.settings = SettingsManager()
        self.library = ModelLibraryManager()
        self.downloader = HFDownloader()
        self.logs = LogManager()
        self.coordinator = EngineCoordinator(logs: logs)
    }

    /// Bootstrap on app launch — must be called once.
    public func bootstrap() async {
        await settings.load()
    }
}

// EngineCoordinator.swift
@Observable
@MainActor
public final class EngineCoordinator {
    public private(set) var status: EngineStatus = .idle
    public private(set) var currentModel: LocalModel?
    public private(set) var engine: (any InferenceEngine)?
    private let logs: LogManager

    public init(logs: LogManager) {
        self.logs = logs
    }

    public func switchTo(_ engineID: EngineID) {
        switch engineID {
        case .mlxSwift: engine = MLXSwiftEngine()
        case .swiftLM, .pythonMLX:
            // v0.1 detection-only — leave engine nil
            engine = nil
        }
    }

    public func load(_ model: LocalModel) async {
        guard let engine else { return }
        do {
            status = .loading(model: model.id)
            try await engine.load(model)
            status = .ready(model: model.id)
            currentModel = model
        } catch {
            status = .error(error.localizedDescription)
        }
    }

    public func unload() async {
        guard let engine else { return }
        try? await engine.unload()
        status = .idle
        currentModel = nil
    }
}
```

Tests: light Xcode-target tests not strictly needed for v0.1 — the underlying actors are tested in MacMLXCore.

```swift
// macMLXApp.swift
@main
struct macMLXApp: App {
    @State private var appState = AppState()

    var body: some Scene {
        WindowGroup {
            RootView()
                .environment(appState)
                .task { await appState.bootstrap() }
        }
        .windowResizability(.contentSize)
    }
}

private struct RootView: View {
    @Environment(AppState.self) private var appState

    var body: some View {
        if appState.settings.current.onboardingComplete {
            MainWindowView()
        } else {
            // Onboarding lands in Task 4
            MainWindowView()  // TEMP — replace with OnboardingWindow once Task 4 is in
        }
    }
}

// MainWindowView.swift
struct MainWindowView: View {
    @Environment(AppState.self) private var appState
    @State private var selectedTab: Tab = .library

    enum Tab: String, CaseIterable, Identifiable {
        case library = "Models"
        case chat    = "Chat"
        case settings = "Settings"
        var id: String { rawValue }
        var systemImage: String {
            switch self {
            case .library: "tray.full"
            case .chat:    "bubble.left.and.bubble.right"
            case .settings: "gear"
            }
        }
    }

    var body: some View {
        NavigationSplitView {
            List(Tab.allCases, selection: $selectedTab) { tab in
                Label(tab.rawValue, systemImage: tab.systemImage)
                    .tag(tab)
            }
            .navigationTitle("macMLX")
            .listStyle(.sidebar)
        } detail: {
            switch selectedTab {
            case .library: ModelLibraryView()
            case .chat:    ChatView()
            case .settings: SettingsView()
            }
        }
        .frame(minWidth: 800, minHeight: 500)
    }
}
```

## Self-Review

**Spec coverage:** Each Surface from the master plan is covered by a Task (1 = foundation, 2 = root, 3-7 = surfaces, 8 = closeout). Logs tab intentionally deferred per UI guidelines.

**No placeholders:** Each task lists exact files, behaviour summary, and links to spec. Code blocks for foundation tasks (1-2) are concrete; surface tasks (3-7) are goal-oriented because SwiftUI patterns are well-known and the spec files are detailed.

**Known fragility:**
- `AppDelegate.swift` for menu bar requires bridging to SwiftUI via `@NSApplicationDelegateAdaptor`. Test before relying.
- `OnboardingWindow.swift` as a separate NSWindow needs careful coordination with the WindowGroup; consider using a Sheet on the main window if NSWindow proves brittle.
- PulseUI is intentionally not added in Stage 4. Adding it post-hoc is a one-step Xcode UI action.
