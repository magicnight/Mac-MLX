# macMLX v0.2 — Overview

> v0.1.0 shipped 2026-04-17. v0.1.1 UX bundle (#1/#2/#3/#17) merged same day.
> This document scopes the v0.2 cycle.

**Goal:** Production-grade UX for download + chat + parameter tuning, plus the remaining engine backends, without expanding the MLX-ecosystem-only scope decided in v0.1.

**Non-goals for v0.2:** VLM (v0.3), benchmark + community leaderboard (v0.3), MAS distribution, iOS port, Linux/Intel support.

## Architecture deltas

| Area | Before (v0.1) | After (v0.2) |
|---|---|---|
| `DownloadProgress` | per-file bytes only | + bytesPerSecond, etaSeconds |
| `HFDownloader` | fire-and-await | task registry + `cancel(modelID:)` API |
| `ChatViewModel` | in-memory messages | persistent `Conversation` JSON at `~/.mac-mlx/conversations/{uuid}.json` |
| Settings | one big struct | + `ModelParameters` per-model overrides at `~/.mac-mlx/model-params/{model-id}.json` |
| `SettingsView` | single form | unchanged; new Inspector panel lives on right side of `MainWindowView` |

## Release plan

- v0.1.1 (optional patch release containing #1/#2/#3/#17): **skip** — roll the same fixes into v0.2.
- v0.2.0 ships once Batch 1 lands green on CI and at least one of Batch 2 items (#19 notarization most valuable).

## Exit criteria for v0.2.0 tag

1. Batch 1 issues closed.
2. `swift test --package-path MacMLXCore` green.
3. `xcodebuild -scheme macMLX build` green.
4. CI green.
5. Signed DMG if Apple Developer enrollment complete; otherwise unsigned DMG with Gatekeeper bypass doc (same as v0.1.0).
6. CHANGELOG `[Unreleased]` section promoted to `[0.2.0] - YYYY-MM-DD`.
