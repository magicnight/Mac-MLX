# macMLX v0.1 — Stage 5: CLI + TUI

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Branch: `feat/stage5-cli-tui`. Tasks build on each other but are largely independent — pick any execution order.

**Goal:** Turn the `macmlx` SPM executable from a stub `--version` printer into a usable developer CLI with 6 subcommands (`serve`, `pull`, `run`, `list`, `ps`, `stop`) plus 3 SwiftTUI dashboards (serve status, pull progress, chat).

**Architecture:** Each subcommand is one `AsyncParsableCommand` struct under `macmlx-cli/Sources/macmlx/Commands/`. They reuse `MacMLXCore`'s `MLXSwiftEngine`, `ModelLibraryManager`, `HFDownloader`, `HummingbirdServer`, `SettingsManager`. Long-running commands (`serve`, `pull`, `run`) detect TTY (`isatty(STDOUT_FILENO)`) and switch between TUI dashboards (SwiftTUI) and plain stdout output.

**Tech Stack:** Swift 6 · `MacMLXCore` (path dep) · `swift-argument-parser` 1.7.1 · `SwiftTUI` (revision-pinned) · `Foundation`.

**Source spec:** `.claude/features/cli-tui.md`.

## Task 2: Shared infrastructure

### CLIContext

```swift
// Sources/macmlx/Shared/CLIContext.swift
import Foundation
import MacMLXCore

/// Bootstrapped per CLI invocation. Loads settings synchronously then exposes
/// the Core actors. Avoids the @MainActor AppState type since CLI doesn't
/// use SwiftUI.
public struct CLIContext: Sendable {
    public let settings: Settings
    public let library: ModelLibraryManager
    public let downloader: HFDownloader

    public static func bootstrap() async throws -> CLIContext {
        let mgr = SettingsManager()
        let s = await mgr.load()
        return CLIContext(
            settings: s,
            library: ModelLibraryManager(),
            downloader: HFDownloader()
        )
    }
}
```

### PIDFile

`~/.mac-mlx/macmlx.pid` stores `{pid, port, model_id?}` so `stop` can find a running `serve` and `ps` can show its status without needing IPC.

```swift
public enum PIDFile {
    public struct Record: Codable, Sendable {
        public var pid: Int32
        public var port: Int
        public var modelID: String?
        public var startedAt: Date
    }

    public static var url: URL {
        FileManager.default.homeDirectoryForCurrentUser
            .appending(path: ".mac-mlx/macmlx.pid")
    }

    public static func write(_ record: Record) throws { /* JSON to atomic */ }
    public static func read() throws -> Record? { /* nil if missing */ }
    public static func clear() throws { /* unlink */ }
}
```

### TTYDetect

```swift
import Darwin

    public static var isInteractive: Bool {
        isatty(STDOUT_FILENO) != 0
    }
```

## Task 3: `macmlx list` (simplest — start here)

```swift
// Commands/ListCommand.swift
import ArgumentParser
import Foundation
import MacMLXCore

struct ListCommand: AsyncParsableCommand {
    static let configuration = CommandConfiguration(
        commandName: "list",
        abstract: "List downloaded models in the configured model directory."
    )

    @Flag(help: "Output as JSON for scripting.") var json = false

    func run() async throws {
        let ctx = try await CLIContext.bootstrap()
        let models = try await ctx.library.scan(ctx.settings.modelDirectory)

        if json {
            let data = try JSONEncoder.prettyPrinted.encode(models)
            FileHandle.standardOutput.write(data)
            print()
        } else {
            // ASCII table: NAME | SIZE | QUANT | MODIFIED
            // (use shell-friendly column widths)
        }
    }
}
```

## Task 5: `macmlx serve` (with TUI)

Boots `MLXSwiftEngine` + `HummingbirdServer`. Optionally auto-loads a model. Writes PIDFile. Cleans up on SIGINT.

```swift
struct ServeCommand: AsyncParsableCommand {
    static let configuration = CommandConfiguration(
        commandName: "serve",
        abstract: "Start the OpenAI-compatible inference server."
    )

    @Option var model: String?
    @Option var port: Int = 8000
    @Option var engine: String = "mlx-swift"
    @Flag(name: .long) var noTui = false

    func run() async throws {
        let ctx = try await CLIContext.bootstrap()
        let engine = MLXSwiftEngine()
        if let modelName = model {
            let local = try await ctx.library.scan(ctx.settings.modelDirectory)
                .first { $0.id == modelName || $0.displayName == modelName }
            guard let local else { throw ValidationError("Model not found: \(modelName)") }
            try await engine.load(local)
        }
        let server = HummingbirdServer(engine: engine)
        let actualPort = try await server.start(preferredPort: port)
        try PIDFile.write(.init(pid: getpid(), port: actualPort, modelID: model, startedAt: Date()))
        defer { try? PIDFile.clear() }

        if TTYDetect.isInteractive && !noTui {
            await ServeDashboard.run(engine: engine, server: server)
        } else {
            print("macmlx serving on http://127.0.0.1:\(actualPort)")
            await Self.waitForSignal()
        }
        await server.stop()
    }
}
```

ServeDashboard renders a SwiftTUI panel showing status / engine / model / port / memory / per-request log lines.

**Commit:** `feat(cli): add serve subcommand with TUI dashboard + PIDFile + signal handling`

## Task 6: `macmlx stop`

Reads PIDFile, sends SIGTERM, waits.

```swift
struct StopCommand: AsyncParsableCommand {
    static let configuration = CommandConfiguration(
        commandName: "stop",
        abstract: "Stop the running macmlx serve."
    )

    func run() async throws {
        guard let record = try PIDFile.read() else {
            print("No macmlx serve running.")
            return
        }
        kill(record.pid, SIGTERM)
        // Wait up to 5s for the PIDFile to be cleaned up.
        for _ in 0..<50 {
            try await Task.sleep(nanoseconds: 100_000_000)
            if (try? PIDFile.read()) == nil { return }
        }
        print("Warning: serve did not exit cleanly.")
        throw ExitCode(2)
    }
}
```

**Commit:** `feat(cli): add stop subcommand sending SIGTERM via PIDFile`

## Task 7: `macmlx pull` (with TUI)

Downloads a model from HF Hub via `HFDownloader`. TUI shows per-file progress.

```swift
struct PullCommand: AsyncParsableCommand {
    static let configuration = CommandConfiguration(
        commandName: "pull",
        abstract: "Download a model from Hugging Face into the model directory."
    )

    @Argument var modelID: String
    @Option var dir: String?

    func run() async throws {
        let ctx = try await CLIContext.bootstrap()
        let target = dir.map { URL(filePath: $0) } ?? ctx.settings.modelDirectory

        if TTYDetect.isInteractive {
            await PullDashboard.run(modelID: modelID, target: target, downloader: ctx.downloader)
        } else {
            // Plain progress to stdout.
            _ = try await ctx.downloader.download(modelID: modelID, to: target)
            print("Downloaded \(modelID) to \(target.path(percentEncoded: false))")
        }
    }
}
```

## Task 8: `macmlx run` (with TUI)

Loads a model, runs single prompt OR interactive chat TUI. Reads from stdin if not a TTY.

```swift
struct RunCommand: AsyncParsableCommand {
    static let configuration = CommandConfiguration(
        commandName: "run",
        abstract: "Run a model interactively (TUI) or with a single prompt."
    )

    @Argument var modelName: String
    @Argument var prompt: String?

    @Option var temperature: Double = 0.7
    @Option(name: .customLong("max-tokens")) var maxTokens: Int = 2048
    @Option var system: String?
    @Flag(name: .customLong("no-stream")) var noStream = false
    @Flag var json = false

    func run() async throws {
        let ctx = try await CLIContext.bootstrap()
        let local = try await ctx.library.scan(ctx.settings.modelDirectory)
            .first { $0.id == modelName || $0.displayName == modelName }
        guard let local else { throw ValidationError("Model not found: \(modelName)") }
        let engine = MLXSwiftEngine()
        try await engine.load(local)

        if let prompt {
            // Single-shot
            try await runSingle(prompt: prompt, engine: engine)
        } else if TTYDetect.isInteractive {
            // Interactive TUI
            await ChatTUI.run(engine: engine, model: local, system: system)
        } else {
            // Stdin REPL
            try await runStdinLoop(engine: engine)
        }
    }
}
```

`runSingle` builds a `GenerateRequest`, streams to stdout, optionally JSON-wraps.

**Commit:** `feat(cli): add run subcommand with single-shot, interactive TUI, and stdin REPL modes`

## Self-Review

**Spec coverage:** All 6 v0.1 subcommands have a Task. 3 TUI dashboards distributed across Serve, Pull, Run. JSON output covered for List and PS.

**Decomposition rationale:** Tasks are mostly independent (commands don't import each other). Only `MacmlxCommand` registration (Task 9) requires all commands to exist first. SwiftTUI dashboards (Tasks 5, 7, 8) can be implemented incrementally — start with non-TUI fallback paths first, then add TUI rendering.

**Known fragility:**
- SwiftTUI on macOS 14+ may have rough edges; Tasks 5/7/8's TUI portions can simplify to plain stdout if SwiftTUI behaves badly. Each TUI dashboard ships with a `--no-tui` escape hatch.
- `kill(pid, 0)` for liveness check assumes the same user owns the process. Sufficient for v0.1.
- `PIDFile.read()` race conditions are not handled; if two `serve` invocations race, only the last one wins. Document but don't fix in v0.1.
