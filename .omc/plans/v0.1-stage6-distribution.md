# macMLX v0.1 — Stage 6: Distribution

> **Sub-plan executor:** Use `superpowers:executing-plans` skill. Branch: `main` (or `feat/stage6-distribution`).
>
> **Critical scope note:** Several tasks here REQUIRE the human (Sparkle key generation is security-sensitive, GitHub Secrets configuration is account-level, the actual `git tag v0.1.0 && push` is a public release event). The autonomous tasks scaffold everything; the manual tasks are clearly flagged.

**Goal:** Tag `v0.1.0`, ship a downloadable DMG via GitHub Releases, with Sparkle auto-update wired to an `appcast.xml` served from the repo.

**Architecture:** GitHub Actions `release.yml` triggered on `v*.*.*` tags builds + archives + DMG-packages + EdDSA-signs the macMLX.app bundle, then commits the updated `appcast.xml` back to `main` and creates the Release with the DMG attached. Sparkle is integrated into the macMLX Xcode target as an SPM dependency.

**Tech Stack:** Swift 6 · `Sparkle` 2.x (EdDSA signatures) · `create-dmg` (already in Brewfile) · GitHub Actions · Python 3.13 (for `update_appcast.py` — user already has `uv` in Brewfile).

**Source spec:** `.claude/distribution.md`.

## Task 1: Add Sparkle SPM dep + AppDelegate updater (AUTONOMOUS)

Since Xcode 16+ uses `PBXFileSystemSynchronizedRootGroup`, adding an SPM dep still requires `.xcodeproj` modification (the dep is registered in project.pbxproj's packageReferences section). Two paths:

**Option A (preferred, AUTONOMOUS):** Edit `project.pbxproj` directly to add the Sparkle package reference and link it to the macMLX target. Risky if the pbxproj structure varies. Verify by `xcodebuild -resolvePackageDependencies`.

**Option B (USER ACTION):** In Xcode → File → Add Package Dependencies → `https://github.com/sparkle-project/Sparkle` → from 2.0.0 → add to macMLX target.

Default to **Option B** with clear instructions. Surface a TODO and continue with downstream code that ASSUMES Sparkle is wired.

Then update `AppDelegate.swift` (already exists from Stage 4) to embed the updater:

        startingUpdater: true,
        updaterDelegate: nil,
        userDriverDelegate: nil
    )
    var updater: SPUUpdater { updaterController.updater }

    func applicationDidFinishLaunching(_ notification: Notification) {
        // Existing menu bar setup + Sparkle starts automatically.

Update `macMLX/macMLX/Info.plist` (or via INFOPLIST_KEY_*) to add:

## Task 3: Build + DMG packaging scripts (AUTONOMOUS)

`scripts/build.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail
APP="macMLX"
VERSION="${GITHUB_REF_NAME#v}"
xcodebuild -project macMLX/macMLX.xcodeproj \
           -scheme "$APP" \
           -configuration Release \
           -archivePath "build/${APP}.xcarchive" \
           archive
xcodebuild -exportArchive \
           -archivePath "build/${APP}.xcarchive" \
           -exportPath "build/export" \
           -exportOptionsPlist scripts/ExportOptions.plist
```

`scripts/ExportOptions.plist` — minimal "developer-id" / "mac-application" export. Since we have no Apple Developer account, use `signingStyle: manual` + empty signing identity:

## Task 4: appcast updater script (AUTONOMOUS)

`scripts/update_appcast.py`:
```python
#!/usr/bin/env python3
"""Prepend a new <item> to appcast.xml for a fresh release."""
import argparse, datetime, sys, xml.etree.ElementTree as ET

def main() -> int:
    p = argparse.ArgumentParser()
    p.add_argument("--version", required=True, help="e.g. 0.1.0")
    p.add_argument("--build", required=True, type=int, help="GITHUB_RUN_NUMBER")
    p.add_argument("--signature", required=True, help="EdDSA signature from sign_update")
    p.add_argument("--size", required=True, type=int, help="DMG file size in bytes")
    p.add_argument("--repo", default="magicnight/mac-mlx")
    p.add_argument("--appcast", default="appcast.xml")
    args = p.parse_args()

    SPARKLE = "http://www.andymatuschak.org/xml-namespaces/sparkle"
    ET.register_namespace("sparkle", SPARKLE)
    tree = ET.parse(args.appcast)
    channel = tree.getroot().find("channel")
    if channel is None:
        print("malformed appcast.xml: no <channel>", file=sys.stderr)
        return 1

    item = ET.Element("item")
    ET.SubElement(item, "title").text = f"Version {args.version}"
    ET.SubElement(item, f"{{{SPARKLE}}}version").text = str(args.build)
    ET.SubElement(item, f"{{{SPARKLE}}}shortVersionString").text = args.version
    ET.SubElement(item, f"{{{SPARKLE}}}minimumSystemVersion").text = "14.0"
    ET.SubElement(item, "pubDate").text = datetime.datetime.now(datetime.UTC).strftime(
        "%a, %d %b %Y %H:%M:%S +0000"
    )
    enclosure = ET.SubElement(item, "enclosure")
    enclosure.set("url", f"https://github.com/{args.repo}/releases/download/v{args.version}/macMLX-v{args.version}.dmg")
    enclosure.set(f"{{{SPARKLE}}}edSignature", args.signature)
    enclosure.set("length", str(args.size))
    enclosure.set("type", "application/octet-stream")
    ET.SubElement(item, f"{{{SPARKLE}}}releaseNotesLink").text = (
        f"https://github.com/{args.repo}/releases/tag/v{args.version}"
    )

    # Insert as the first <item> after <description>/<language>
    insert_index = 0
    for i, child in enumerate(channel):
        if child.tag == "item":
            insert_index = i
            break
        insert_index = i + 1
    channel.insert(insert_index, item)

    ET.indent(tree, space="  ")
    tree.write(args.appcast, encoding="utf-8", xml_declaration=True)
    print(f"appcast.xml updated: v{args.version} (build {args.build}, {args.size} bytes)")
    return 0

if __name__ == "__main__":
    sys.exit(main())
```

## Task 6: Sparkle EdDSA key generation (USER ACTION — flagged)

The user must:
1. Run `~/Library/Developer/Xcode/DerivedData/.../SourcePackages/checkouts/Sparkle/bin/generate_keys` (after Sparkle is installed via SPM in Task 1) OR `brew install sparkle && /opt/homebrew/Caskroom/sparkle/<ver>/bin/generate_keys`
2. Paste the **public key** into `Info.plist`:
   ```xml
   <key>SUPublicEDKey</key>
   <string>BASE64_PUBLIC_KEY_HERE</string>
   ```
3. Save the **private key** to `~/.ssh/sparkle_private_key.pem` (or wherever) — **never commit this**.
4. Copy the contents of the private key file into a GitHub repository Secret named `SPARKLE_PRIVATE_KEY` at https://github.com/magicnight/Mac-MLX/settings/secrets/actions

The autonomous parts of this stage do everything except the actual key creation + secret setting.

## Task 7: First release dry run (USER ACTION + autonomous prep)

Before the real `git tag v0.1.0`:

1. **Local DMG smoke**: run `scripts/build.sh && scripts/package-dmg.sh` locally; manually open the DMG, drag to /Applications, right-click → Open → Open the app, verify it launches and the menu bar icon appears.
2. **GitHub Actions billing**: must be restored before tagging — release.yml requires `macos-15` runners.
3. **Sparkle key in place**: Task 6 done.
4. **Tag**: `git tag v0.1.0 -m "macMLX v0.1.0 — first release" && git push --tags`
5. **Watch the workflow**: it builds, signs, packages, updates appcast, creates the GitHub Release.
6. **Smoke install**: download the DMG from the Release page on a different Mac (or after deleting the local copy), confirm it installs.

## Self-Review

**Spec coverage:** Every item in `.claude/distribution.md` § Sparkle, DMG, CI is addressed. The autonomous portion handles all scaffolding.

**Hard limits without user:** EdDSA key generation, GitHub Secret configuration, public release tagging, and DMG smoke install all require human action — security and account-level operations.

**Known fragility:**
- The `Info.plist` modifications may need to use `INFOPLIST_KEY_*` build settings instead of a separate file, depending on how the Xcode target is configured. Check first; if generated, edit project.pbxproj's INFOPLIST_KEY_* lines.
- macOS 14.0+ deployment target restricts users; `LSMinimumSystemVersion` already 14.0 from Stage 1.
- Without an Apple Developer account, the DMG is unsigned; first-launch requires the documented Right-click → Open Gatekeeper bypass (already in README).
