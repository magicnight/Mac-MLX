## 1. Version string — auto-bump via release workflow

### Root cause
`macmlx --version` reads `MacMLXCore.version`, declared as:

```swift
// MacMLXCore/Sources/MacMLXCore/MacMLXCore.swift:7
public static let version = "0.1.0"
```

Hard-coded at v0.1 creation; **never updated**. CLI therefore
reports `0.1.0` no matter the tag.

### Fix: option C (automated sed injection)

Mirror the pattern `release.yml` already uses for Xcode's
`MARKETING_VERSION`. Add a step **before** `Build & archive`:

```yaml
- name: Bump MacMLXCore.version
  env:
    GITHUB_REF_NAME: ${{ github.ref_name }}
  run: |
    VERSION="${GITHUB_REF_NAME#v}"
    sed -i '' \
      "s/public static let version = \".*\"/public static let version = \"$VERSION\"/" \
      MacMLXCore/Sources/MacMLXCore/MacMLXCore.swift

    # Sanity check — fail the run rather than publish an unbumped build.
    grep -q "public static let version = \"$VERSION\"" \
      MacMLXCore/Sources/MacMLXCore/MacMLXCore.swift \
      || { echo "error: version sed did not apply"; exit 1; }
```

### Trade-offs

| | Option A hand-edit | Option B SPM plugin | **Option C sed (chosen)** |
|---|---|---|---|
| Failure mode | Forget to bump | Complex to author | CI step fails loudly if sed misses |
| Maintenance | Every release | Write once | Low — release.yml already does this for MARKETING_VERSION |
| Reversibility | Manual revert | Edit plugin | Edit workflow YAML |

Tag-time source ends up differing by one line from `main`, but that's
fine — the checked-out commit on the tag ref is ephemeral to the
workflow. Optional: push the `sed`-modified file back to main via the
same rebase+retry pattern the appcast push uses. Probably overkill
for a constant that changes every release anyway.

### Files touched
- `.github/workflows/release.yml`
- `MacMLXCore/Sources/MacMLXCore/MacMLXCore.swift` (value itself becomes irrelevant — CI overwrites)

## 2. `macmlx search <query>` command

### Behaviour

```
$ macmlx search qwen3
NAME                                        DOWNLOADS    LIKES
mlx-community/Qwen3-8B-4bit                    12,847      142
mlx-community/Qwen3-14B-4bit                    8,320       89
mlx-community/Qwen3-32B-4bit                    3,102       54
…  (10 rows by default)

$ macmlx search qwen3 --limit 20
$ macmlx search qwen3 --sort likes          # or downloads (default) / recent
$ macmlx search qwen3 --json                # structured for scripting
```

### Design decisions (confirmed 2026-04-17)

- **Default limit**: `10`
- **Default sort**: `downloads` descending
- **User-configurable**: `--limit`, `--sort`
- **Author filter**: hard-coded to `mlx-community` (matches GUI Hugging
  Face tab behaviour; don't expose `--author` flag — adds complexity
  for the ~2% of users who'd want it)
- **`--json`**: flag mirrors other commands

### Implementation sketch

- New `macmlx-cli/Sources/macmlx/Commands/SearchCommand.swift`
- Backend: reuse `CLIContext.downloader.search(query:limit:)` —
  already exists from v0.1, returns `[HFModel]`, honours `hfEndpoint`
  mirror setting
- Table formatter: reuse `padLeft` / `padRight` helpers from
  `ListCommand` (extract to a shared `CLITable.swift` if both grow
  further)
- Register in `MacmlxCommand.subcommands`

### Scope of `--sort`

`HFDownloader.search` currently passes `sort=downloads direction=-1`
to the HF API. Expose `--sort` as a flag with three values:
- `downloads` (default)
- `likes`
- `recent` (maps to HF's `lastModified`)

If HF API rejects an unknown sort key, fall back to `downloads`.

## 3. Binary slim-down

Current release binary: ~60 MB. User confirms target newest macOS
only — acceptable to drop backward-compat to shrink further.

### Measures (confirmed 2026-04-17)

| Measure | Expected saving | Cost |
|---|---|---|
| **Strip symbols** (`strip -S`) | ~5–10 MB | Loses DWARF; release-only |
| **Dynamically link Swift stdlib** | ~5–8 MB | Requires user's macOS to have matching Swift runtime — already standard on macOS 14+ |
| `-Xlinker -dead_strip` | usually ~0 (release defaults on) | — |

Target: ~40–45 MB binary. Still linked-heavy thanks to MLX runtime,
but ~30% smaller.

### Pulse logging — **exposed, not removed**

User note: "不能白做这个功能" — Pulse stays, and v0.3.6 adds a CLI
flag so logs are actually visible from the command line:

```
$ macmlx serve --log-level debug
$ macmlx run model "prompt" --log-level info --log-stderr
```

**Design:**
- `--log-level` (global flag, `debug` / `info` / `warning` / `error` / `critical` / `off`; default `warning`)
- `--log-stderr` flag: when set, tee log messages to stderr in plain text in addition to the Pulse store. Makes runtime debugging friction-free without opening the GUI Logs tab.
- Implementation: `CLIContext` stores a `LogLevel` + `teeToStderr` bool. Replaces LogManager's default store usage in CLI-only flows with a filtered wrapper.
